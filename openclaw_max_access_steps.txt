OpenClaw maximum practical access (non-root)

Important
- True full device control requires root.
- With Termux + Shizuku, you can still grant broad shell-level control.
- Run these on your Android device in Termux (not on Windows).

1) Basic Termux setup
pkg update -y
pkg install -y android-tools nmap termux-api
termux-setup-storage

2) Start and verify Shizuku bridge
shizuku
rish -c whoami

3) Keep Termux alive in background (best effort)
rish -c "dumpsys deviceidle whitelist +com.termux"
rish -c "cmd appops set com.termux RUN_IN_BACKGROUND allow"
rish -c "cmd appops set com.termux RUN_ANY_IN_BACKGROUND allow"
rish -c "cmd appops set com.termux WAKE_LOCK allow"

4) Grant storage and media permissions to Termux (best effort)
rish -c "pm grant com.termux android.permission.READ_EXTERNAL_STORAGE" || true
rish -c "pm grant com.termux android.permission.WRITE_EXTERNAL_STORAGE" || true
rish -c "pm grant com.termux android.permission.READ_MEDIA_IMAGES" || true
rish -c "pm grant com.termux android.permission.READ_MEDIA_VIDEO" || true
rish -c "pm grant com.termux android.permission.READ_MEDIA_AUDIO" || true

5) Optional: disable battery optimization manually
- Android Settings -> Apps -> Termux -> Battery -> Unrestricted
- Android Settings -> Apps -> Shizuku -> Battery -> Unrestricted

6) OpenClaw post-install commands
openclaw onboard
openclaw auth add google --key YOUR_GEMINI_KEY
openclaw gateway

7) Quick verification
command -v openclaw
rish -c id
rish -c "settings get global adb_wifi_enabled"

If a grant fails
- Some permissions are Android-version dependent and may fail safely.
- Keep using rish -c for privileged shell actions where direct grant is blocked.
